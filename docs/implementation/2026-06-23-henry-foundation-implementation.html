Implemented Scope

The Foundation layer now defines the shared contracts that later workstreams depend on. This includes project metadata, app branding, environment settings, typed Slack/channel DTOs, agent run contracts, sandbox request/response contracts, runtime-checkable protocols, database models and migrations, channel configuration loading, package-scanned integration discovery, and deterministic testing fakes.

Foundation Tasks Covered

Plan task Implementation details
F1 Repo init pyproject.toml now pins pydantic-ai==2.0.0; uv.lock records resolved dependency versions. Existing AGPL license, CLA, compose file, README, and package scaffold are present.
F2 Branding and settings henry/branding.py centralizes the application name. henry/settings.py defines Settings with the HENRY_ prefix, .env support, model, Slack, GitHub, web search, LiteLLM, budget, and sandbox fields.
F3 Core types and contracts henry/types.py provides channel context, memory, state, message, and transcript DTOs. henry/contracts.py provides AgentDeps, ToolSpec, run usage/result DTOs, AgentRunner, and SlackEvent.
F4 Interfaces henry/interfaces.py defines runtime-checkable Memory, Sandbox, and Integration protocols. henry/sandbox_types.py defines immutable sandbox policy and exec request types plus exec results.
F5 Database and migrations henry/db/models.py defines SQLAlchemy 2 models with deterministic naming conventions, JSONB-backed JSON columns, timezone-aware timestamps, and required indexes. henry/db/session.py exposes factory functions only. alembic.ini and henry/db/migrations/versions/20260623_0001_foundation.py establish the reviewed baseline migration.
F6 Channel config registry henry/config/defaults.yaml stores JSON-shaped YAML defaults. henry/config/registry.py deep-merges defaults with a DB/fake row, forbids unknown config keys, validates model strings, and optionally validates enabled integrations against a known registry.
F7 Integration registry henry/integrations/registry.py scans henry.integrations.builtins, imports modules, calls get_integration(), and rejects duplicate integration names. This lets future workstreams add built-in integrations without editing shared registry files.
F8 Fakes henry/testing/fakes.py provides channel-scoped FakeMemory, deterministic call-recording FakeSandbox, a real async echo-tool FakeIntegration, and canned FakeAgentRunner.
